# 木炉星篝火 new tab — 产品需求文档

> 版本 1.2.0 · 与代码实现同步

## 产品定位

- **载体**：Chrome MV3 扩展，覆盖新标签页
- **体验**：Outer Wilds 风格 3D 篝火场景 + 沉浸式时钟 + 可选功能模块
- **原则**：用户无交互时，篝火和时钟保持绝对视觉主导；工具只在用户主动打开时出现
- **V1.1 主更新**：底部启动坞、模块开关框架、今日 TODO

## 功能模块框架

- 功能以模块注册表管理，当前模块为：
  - `pomodoro`：番茄钟
  - `todayTodo`：今日 TODO
  - `quickLinks`：快捷导航
  - `readLater`：稍后再看
- 默认启用番茄钟、今日 TODO、快捷导航与稍后再看。
- 设置工作台提供模块开关；至少保留一个模块启用，避免启动坞为空。
- 模块设置持久化到 `chrome.storage.sync` 的 `featureSettings`。
- `featureSettings` 包含启用模块列表与最近打开模块，为后续搜索、快捷操作等模块预留。

## 底部启动坞与观测舱面板

- 页面底部显示低调启动坞，只展示已启用模块的统一风格 SVG 图标。
- 图标采用星图仪器感线性 SVG：统一线宽、圆角端点、微弱发光，不使用 emoji 或单字短标签。
- 鼠标移入模块图标时，进入 hover preview：
  - 对应功能面板从右侧缓慢滑入。
  - 3D canvas 始终铺满全屏，不缩成卡片、不露出页面底色。
  - Three.js 相机与观察目标轻微平移，让篝火构图向左让位给右侧功能区。
  - 鼠标在启动坞或面板内移动时，预览保持打开。
- 鼠标移出启动坞与面板时，如果没有常驻模块，面板向右收回，相机恢复初始构图。
- 点击模块图标时，进入 pinned state：
  - 当前功能面板常驻，场景保持让位状态。
  - 点击另一个模块切换常驻模块。
  - 再次点击同一个模块或按 `Escape` 取消常驻并关闭面板。
- 右上角齿轮保留为全局设置入口；设置以右侧全高工作台打开，而非居中小弹框。

## 设置工作台

- 设置入口从右上角齿轮打开，界面从右侧滑出为全高设置工作台。
- 桌面端采用“左侧分区导航 + 右侧设置内容”的结构，承载功能模块、快捷导航、番茄钟等多组设置。
- 窄屏端设置工作台全屏展示，分区导航横向滚动，设置内容纵向滚动。
- 设置工作台打开期间，底部启动坞隐藏并禁用交互，避免与设置操作争夺焦点。
- `Escape`、点击遮罩或关闭按钮可关闭设置工作台。
- 设置工作台仍为临时层，不改变默认无交互时的沉浸式篝火主画面。
- UI 通过 `setDockInteractionState` 将 `closed` / `preview` / `pinned` 状态传给 3D 场景，当前版本做轻量火光与运动联动，后续可扩展为相机/景深联动。

## 番茄钟

### 核心流程

1. **Idle**：显示当前时间，番茄钟模块抽屉中可开始专注
2. **Focus**：弱化时钟，显示 MM:SS 倒计时，可暂停/放弃
3. **专注完成**：立即进入烤火近景，同时开始短/长休息倒计时
4. **ShortBreak / LongBreak**：休息倒计时，可跳过；可手动退出烤火视角
5. 长休息结束轮次归零；可选自动开始下一轮

### 轮次规则

- `round` 始终为 **1 ~ cyclesBeforeLong** 的整数，表示当前周期内第几次专注
- 第 1 ~ (N−1) 次专注结束 → 短休息 → 轮次 +1
- 第 N 次专注结束 → 长休息（轮次保持 N）→ 长休息结束后轮次重置为 1
- 放弃专注、修改「几轮后长休息」、加载历史会话时自动校正非法轮次，避免出现「第 5/4 轮」

### 默认设置

| 参数 | 默认值 | 范围 |
|------|--------|------|
| 专注 | 25 min | 5–90 |
| 短休息 | 5 min | 1–30 |
| 长休息 | 15 min | 5–45 |
| 长休息轮数 | 4 | 2–8 |

## 今日 TODO

- 今日 TODO 表示当前待办清单，不按日期自动校验、迁移、归档或删除。
- TODO 数据持久化到 `chrome.storage.local` 的 `todayTodoItems`。
- Item 字段：
  - `id`
  - `title`
  - `completed`
  - `createdAt`
  - `updatedAt`
  - `completedAt`
- 支持快速添加、完成/取消完成、编辑标题、删除单条、清空已完成。
- 添加区仅保留任务输入与添加按钮，任务项保持轻量，长标题可自动换行并完整编辑。
- 不支持历史 TODO 视图、提醒、日期归档、番茄钟任务绑定。

## 快捷导航

- 快捷导航用于承载常用网站入口，是 V1.1 模块框架下的第三个功能模块。
- 设置工作台提供快捷导航管理区，支持添加、删除和通过拖动柄调整网站顺序。
- 添加网站时：
  - 用户输入网址，名称可选。
  - 未输入协议时自动补全为 `https://`。
  - 仅接受 `http` / `https` URL。
  - 名称为空时根据域名自动生成短名称。
  - 自动推导 favicon 地址为站点根路径 `/favicon.ico`，用于生成圆形图标。
- favicon 加载失败时，圆形图标显示站点名称首字作为兜底。
- 快捷导航数据持久化到 `chrome.storage.sync` 的 `quickLinks`，最多保留 24 个链接；每个链接保存 `position` 用于展示顺序。
- 快捷导航面板以圆形图标网格展示，按用户在设置工作台拖动后的顺序排列，点击后在新标签页打开目标网站。
- V1.1 不支持分类、拖拽排序、批量导入、手动上传 icon、搜索建议或访问历史排序。

## 稍后再看

### 背景与目标

- 用户浏览文章或视频时，可点击浏览器工具栏中的扩展图标，一次点击保存当前网页。
- 用户随后在新标签页底部启动坞打开「稍后再看」，缩短从收集到再次打开的路径。
- 提醒必须保持克制：默认不展示数量、不自动打开面板、不发送系统通知；仅在悬浮「稍后再看」Dock 图标及对应面板交互期间显示待消费数量。
- 本模块是临时内容消费队列，不替代笔记软件；V1.2 不包含标签、文件夹、备注、全文抓取、搜索、云同步或积压提醒。

### 保存入口与反馈

- `manifest.json` 声明 Chrome MV3 `action`，点击工具栏图标后由 `background.js` 保存当前活动标签页。
- 仅接受 `http` / `https` 页面；浏览器设置页、扩展页、新标签页、本地文件等特殊页面不创建记录。
- 保存动作不打开 popup、新标签页或管理面板。
- 工具栏图标使用短暂徽标反馈，约 1.8 秒后清除：
  - 首次保存：绿色 `✓`，提示「已加入稍后再看」。
  - 链接已存在，或已完成内容被重新保存：显示「已存」，统一提示「已在稍后再看中」。
  - 保存失败：红色 `!`，悬浮说明展示失败原因。
- URL 去重：
  - 未开始或正在看的重复链接不新增、不重置消费状态。
  - 已完成链接再次保存时恢复为未开始、更新保存时间，并回到待消费列表顶部。
  - 重复保存时允许刷新标题和 favicon。

### 数据模型与持久化

- 数据持久化到 `chrome.storage.local` 的 `readLaterItems`，不随 Chrome 账号同步。
- 排序偏好持久化到 `chrome.storage.local` 的 `readLaterSettings`。
- Item 字段：
  - `id`
  - `url`
  - `title`
  - `faviconUrl`
  - `contentType`：`article` / `video`
  - `status`：`unread` / `inProgress` / `completed`
  - `createdAt`
  - `openedAt`
  - `completedAt`
- `openedAt` 记录第一次从面板打开的时间，V1.2 不要求在界面展示。

### 分类、状态与排序

- 首次保存时按已知视频站点自动识别视频，其余页面默认归为文章。
- 主列表同时展示「文章」与「视频」两个真实分组，不采用单选分类筛选。
- 用户通过条目拖动柄将链接拖入另一分组；目标分组在拖动期间高亮，空分组仍保留可放置区域。
- 窄屏或无法拖动时，通过条目菜单提供「移至文章 / 移至视频」兜底操作。
- 消费状态为三段式：
  - `unread`：保存后的默认状态。
  - `inProgress`：用户从面板打开链接后自动进入；再次打开保持不变。
  - `completed`：仅由用户主动标记，代表确认完成内容消费。
- 用户可将未开始或正在看的条目直接标记完成；已完成条目可恢复为正在看。
- 文章和视频主分组只展示未开始与正在看的内容；已完成内容默认折叠在底部，展开后仍按文章、视频区分。
- 默认按保存时间倒序排列，可切换为正序；偏好持久化，且分别作用于各分组。已完成内容仍按保存时间排序。

### 打开、提醒与删除

- 点击条目主体在当前新标签页打开，并把未开始状态更新为正在看。
- 条目右侧提供可发现的「在新标签页打开」按钮；`Ctrl` / `Cmd` + 点击及鼠标中键仍遵循浏览器习惯。
- 点击拖动柄、完成按钮、更多菜单时不得误触打开网页。
- Dock 待消费数量 = 未开始数量 + 正在看数量；数量为 0 时不显示。
- 数量气泡只在悬浮「稍后再看」Dock 图标以及鼠标继续移动到对应预览面板期间显示，面板完全收起后隐藏。
- 所有状态下的条目均可删除：
  - 单条删除立即执行，面板底部提供约 5 秒撤销，不增加回收站。
  - 「管理」进入多选模式，复选框仅在该模式显示，支持全选当前可见范围。
  - 批量删除前统一确认，确认后不提供批量撤销。
- 条目采用紧凑布局：favicon、最多两行标题、来源域名、保存日期、状态、完成操作与更多菜单；favicon 失败时以域名首字母兜底。

## 音频控制

- 专注阶段显示声音按钮与音量条；按钮仅打开/关闭篝火白噪音，不影响阶段提示音和休息背景音乐。
- 音量条悬浮出现，允许鼠标从按钮移动到滑条；离开控件后短暂停留并自动收回。
- 篝火白噪音静音状态持久化到 `localStorage` 的 `pomodoroAudioMuted`；音量持久化到 `pomodoroAudioVolume`。
- 「专注/休息开始提示音」「专注篝火白噪音」「休息背景音乐」在设置中分别控制，互不替代。

## UI 状态

| 状态 | 时钟 | 主显示 | 工具入口 |
|------|------|--------|----------|
| Idle | 100% | 当前时间 | 底部启动坞 |
| Focus | 35% | 倒计时 | 底部启动坞 + HUD 操作 |
| Paused | 35% | 冻结时间 | 底部启动坞 + HUD 操作 |
| Short/Long Break | 35% | 倒计时（可叠加烤火近景） | 底部启动坞 + HUD 操作 |

## 技术架构

```
newtab.html → newtab-main.js
  ├── feature-settings.js（模块注册表 + 模块开关 storage）
  ├── todo.js（今日 TODO 数据控制器 + storage）
  ├── quick-links.js（快捷导航数据控制器 + favicon 推导）
  ├── read-later.js（稍后再看数据、状态、分类与排序控制器）
  ├── pomodoro.js（状态机 + storage）
  ├── pomodoro-ui.js（HUD / SVG 启动坞 / 右侧观测舱面板 / 设置工作台 / TODO / 快捷导航 UI）
  ├── confirm-dialog.js（统一确认弹窗）
  ├── audio.js（专注白噪音 + MP3 提示音 + 休息背景音乐）
  ├── app.js（Three.js 场景）
  └── background.js（alarms + notifications + 工具栏一键保存）
```
